# Initializations

CONFIG-->MQSF BASE: Provides simple config
PLUGIN IMPLEMENTATION-->MQSF BASE: provides hook implementation
note over MQSF BASE: Initializes
MQSF BASE-->INPUT MESSAGE QUEUE: Sets listener on input MQ
SERVICE CLIENT-->OUTPUT MESSAGE QUEUE: Starts listening on results queue
SERVICE CLIENT->INPUT MESSAGE QUEUE:Publishes service request\n (includes ID)
INPUT MESSAGE QUEUE->MQSF BASE: Gets the service request
MQSF BASE-->PLUGIN IMPLEMENTATION: Invokes plugin logic
box over PLUGIN IMPLEMENTATION: Plugin logic gets exec
PLUGIN IMPLEMENTATION-->MQSF BASE: returns result data
MQSF BASE->OUTPUT MESSAGE QUEUE:Publishes service result\n(includes ID)\n
OUTPUT MESSAGE QUEUE->SERVICE CLIENT:Receives service result\n(includes ID)
box over SERVICE CLIENT: Correlates its response \nwith the request \nusing ID and continues with\nservice logics
